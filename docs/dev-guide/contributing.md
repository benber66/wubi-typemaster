# 贡献指南

> 欢迎贡献 WubiTypeMaster！

## 1. 行为准则

请保持友好和专业的讨论环境。尊重所有贡献者。

## 2. 如何贡献

### 2.1 报告 Bug

在 GitHub Issues 提交，包含：

- 复现步骤
- 预期 vs 实际
- 截图（如果相关）
- 系统信息（OS、Node 版本、IME）

### 2.2 提议功能

先在 Discussions 讨论，再决定是否开 Issue。

### 2.3 提交代码

1. Fork 仓库
2. 创建特性分支
3. 编码 + 测试
4. 推送并开 PR

## 3. PR 流程

1. 确保 CI 全绿
2. 至少 1 位维护者 review
3. Squash 合并

## 4. 五笔码表贡献

如果你发现五笔码表错误：

1. 提供：汉字、你认为正确的码、出处
2. 在 Issue 中标注 `data-correction`
3. 维护者验证后合并

## 5. 文档贡献

文档不全？用词不当？欢迎直接 PR。`docs/` 下的 Markdown 文件就是源代码。

## 6. 本地化

i18n 暂未实现，但文案集中管理后欢迎翻译贡献。

## 7. 联系方式

- GitHub Issues / Discussions
- 邮箱：见 GitHub profile
